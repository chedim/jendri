Jendri configuration

To change Jendri behavior and integrate it in your project easily, you can change some configuration values in the global object called "Jendri". In example, here is a part of this website index.html: 

    <HEAD>
        <script type="text/javascript">
            window.Jendri = {
                source: "/jendri/",
                container: '#main_content',
                debug: true,
                siteVersion: '1.6',
                interface: 'docs',
                home: 'docs'
            }
        </script>
        <script type="application/javascript" src="/jendri/lib/jquery-2.1.1.min.js"></script>
        <script type="application/javascript" src="/jendri/services/jendri.js"></script>

All of configuration variables are listed here:

  • source — relative url of Jendri files location (default: "")
  • startup — startup service (default: "router")
  • home — default index page (default: "index")
  • debug — console debug level (default: false)

    Possible values:

    • <true|"on"> - console calls are enabled, calls not supported by a browser are silently ignored
    • <false|"off"> - all console calls are silently ignored
    • <"log"|"debug"|"info"|"warn"|"error"> - all console calls are disabled except logging calls with a logging level that is equal or greater than the provided threshold

  • container — selector of an element that Jendri will use as an interfaces html-container (default: "body")
  • siteVersion — version of the site, used by appcache service to determine cache validity (default: undefined — disable cache)
  • interface — name of the interface that will be loaded as default (default: "default")
