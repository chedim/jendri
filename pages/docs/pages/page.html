Jendri pages

Jendri pages are almost the same as widgets, except for several moments:

  • At any moment there can be only one page running
  • The running page is determined by the current url value
  • All pages are located under the "page" directory

To call a page, at your browser address bar append the pound sign and the page address (relative to the "pages" directory): 

    http://yourwebsite.com/#article
    http://yourwebsite.com/#register

You can pass some arguments to the page that you want to load: 

    http://yourwebsite.com/#article?id=97465
    http://yourwebsite.com/#register?role=admin

If you will change arguments inside of the address bar, loaded page will get the "arguments" event as described on the "widgets page

To load any page from your code, use the 'router.navigate' function or any other url changing method: 

    me.router = 'router';
    $(c).on('resume', function() {
        me.router.navigate('profile');
        // or
        window.location.hash = 'profile';
    }

Interfaces

Interfaces wrap any loaded page and provide some useful and static website elements: menu, sidebars, etc. Technically interfaces realization is the same as widgets and pages, but you can't embed them or call them from the browser address bar. All interfaces are located under the 'interfaces' directory.

To change current interface, use the "router.setUI" method:

    me.router = 'router';
    $(c).on('resume', function() {
        if (window.user) {
            if (window.user.admin) {
                Router.setUI('admin');
            } else {
                Router.setUI('user');
            }
        } else {
            Router.setUI('guest');
        }
    }

It's always a better idea to have a separate website for your admin-panel ;-)
