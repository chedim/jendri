Jendri widgets

Widgets in Jendri are the main website building blocks. Usually they responsible of the contents, look and behavior of their html-container contents, so they consist of three files:

  1. page.html — html-template, that will be injected inside of the html-container before widget will be started.
  2. style.less — less-code, that will affect how the html-container contents will look.
  3. code.js — javascript code that will create widget's behavior.

To create a widget, create a subdirectory in the "widgets" directory and place there at least one of the "page.html" or "code.js" files: 

    /widgets/
        menu/
            page.html
        picker/
            date/
                code.js
                page.html
                style.less

To inject a widget, set the "widget" attribute of any html component to the widget address in the "widgets" directory: 

    <div widget="menu"/>
    <div widget="picker/date"/>

Widget lifecycle

During the widget lifecycle, Jendri Router emits some events on it's html-container element:

  • start — when the widget and all it's required dependencies had loaded
  • resume — when the widget's html-container becomes visible and available for user
  • pause — when the widget's html-container is no longer visible or available for user
  • free — when the widget should be unloaded
  • arguments — when any new arguments had been passed to the widget through it's address

Widget arguments

There is several ways to pass some arguments to the widget: through it's html-container, through it's address or through it's process object.

Html-container arguments

First of all, when it starts, your widget can check some attributes on it's html-container. But also you can pass some html-code to the widget, setting it's html-container contents before the widget is loaded. When Jendri will load the widget's template file, it will append contents of the container to the "content" tag: 

    <-- this is an example of widget html-template that able to hold passed html-content: -->
    Hello! Your passed html content is: <content/>

Address arguments

When you inserting a widget, you can pass some arguments to it as you pass arguments at HTTP GET-requests: 

    <div widget="calendar?month=september"/>

In this case after the Calendar widget will be loaded, Jendri will emit the 'arguments' event to the widget with the arguments that you declared in the widget address.

Process arguments

Your widget can provide some interface to control it through it's process object: 

    me.setMonth = function(month) { /* ... */ };

To access this interface, get the widget process object and call any of widget's declared public functions: 

    $(month_widget_container).process().setMonth('september');
